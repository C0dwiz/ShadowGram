# 👁️ Накрутка просмотров (Channel Viewer)

Этот модуль предназначен для массового просмотра постов в Telegram-каналах. Его главной особенностью является **полная параллельность**: все выбранные аккаунты начинают работу одновременно, что значительно ускоряет процесс накрутки «глазиков» на постах.

### 📝 Основные функции
1.  **Параллельный запуск**: Модуль использует `asyncio.gather`, позволяя десяткам аккаунтов работать синхронно без задержек друг на друга.
2.  **Staggered Start (Разнесенный старт)**: Чтобы аккаунты не заходили в сеть одновременно, каждый из них выбирает случайное время ожидания от 1 до 120 минут перед началом первого цикла.
3.  **Бесконечные ротации**: После завершения просмотра всех постов аккаунт засыпает на 3–6 часов, после чего автоматически повторяет задачу.
4.  **Эмуляция живого присутствия**: Перед переходом в целевой канал бот просматривает список своих чатов (`get_dialogs`), создавая видимость естественного входа в приложение.
3.  **Умное чтение постов**:
    *   Сбор последних сообщений (до указанного лимита).
    *   Рандомизация порядка просмотра (каждый аккаунт «читает» посты в своем порядке).
    *   Просмотр блоками (чанками) по 2–5 постов.
4.  **Низкоуровневая накрутка**: Использует функцию API `GetMessagesViews`, которая официально засчитывает просмотр («глазик») на серверах Telegram.
5.  **Отметка о прочтении**: Помечает историю чата как прочитанную после каждого блока сообщений.

### ⚙️ Параметры модуля
- **Ссылка на канал**: Публичная ссылка (например, `@username`) или ID канала. Аккаунты должны быть подписаны на этот канал для корректной работы.
- **Лимит постов**: Количество последних сообщений, которые нужно просмотреть. По умолчанию — 60.

### 💡 Советы по использованию
- **Прокси**: При параллельном запуске крайне важно использовать индивидуальные прокси для каждого аккаунта, чтобы Telegram не видел массовую активность с одного IP.
- **Подписка**: Убедитесь, что ваши аккаунты подписаны на канал. Если аккаунт не является участником, просмотры могут не засчитываться или засчитываться с низким приоритетом.
- **Задержки**: Между блоками сообщений модуль делает паузы 10–25 секунд. Это необходимо для имитации вдумчивого чтения контента.

### 🛡️ Безопасность
Благодаря нашей архитектуре, каждый аккаунт в параллельном режиме работает в своей изолированной среде (свой порт прокси, своя сессия), что делает процесс максимально безопасным.
